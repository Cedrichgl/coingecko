CoinGecko API → ETL (fetch + upsert) → PostgreSQL → FastAPI

→ API Externe : Récupération et nettoyage de 5 000 cryptomonnaies avec gestion des rate limits

→ Sqlalchemy : Modélisation des données avec l'ORM et gestion des conflits via upsert

→ Postgresql : Stockage structuré et persistant des données

→ FastAPI : Exposition des données via des endpoints REST

→ Alembic : Versioning des migrations de base de données

Objectif : Créer le socle d'une décision data-driven en transformant une source brute en données exploitables, accessibles et maintenables.
